# Lab3
### LinkedBlockingDeque

LinkedBlockingDeque — это реализация блокирующей двусторонней очереди (deque) на основе связанного списка, которая поддерживает многопоточность. Она позволяет добавлять и удалять элементы с обоих концов очереди, при этом обеспечивая блокировку потоков при необходимости.  

### Основные компоненты
- LinkedList: Внутренний связанный список, который хранит элементы очереди.
- Lock: Объект блокировки, который используется для синхронизации доступа к очереди.
- Condition: Условия, которые используются для управления ожиданием и уведомлением потоков. В LinkedBlockingDeque используются два условия: notEmpty и notFull.
- Capacity: Максимальное количество элементов, которое может содержать очередь.

### Основные методы
- addFirst(E e): Добавляет элемент в начало очереди.
- addLast(E e): Добавляет элемент в конец очереди.
- removeFirst(): Удаляет и возвращает элемент из начала очереди.
- removeLast(): Удаляет и возвращает элемент из конца очереди.
- peekFirst(): Возвращает, но не удаляет элемент из начала очереди.
- peekLast(): Возвращает, но не удаляет элемент из конца очереди.
- size(): Возвращает количество элементов в очереди.
- isEmpty(): Проверяет, пуста ли очередь.
- isFull(): Проверяет, полна ли очередь.

### Алгоритм работы: 
Добавление элементов
Когда поток пытается добавить элемент в очеред, он вызывает метод addFirst или addLast.

Поток зхватывает блокировку.
Идет проверка, полна ли очередь. Если очередь полна, поток вызывает notFull.await(), чтобы освободить блокировку и перейти в состояние ожидания.
Если очередь не полна, поток добавляет элемент в начало или конец очереди.
Поток вызывает notEmpty.signal(), чтобы уведомить один из ожидающих потоков, что очередь теперь не пуста.
Поток освобождает блокировку.

Удаление элементов работает аналогично, только идет проверка что очередь не пуста.

### Вывод
LinkedBlockingDeque показала высокую эффективность в многопоточной среде благодаря использованию блокировок и условий. Время выполнения операций зависит от емкости очереди и количества потоков: при малых объемах операции выполняются быстро, но с увеличением нагрузки время выполнения возрастает. Особенно заметна зависимость производительности у операций удаления и чтения элементов, где задержки увеличиваются при росте емкости и количества потоков. Очередь демонстрирует надежность и стабильность в обработке данных при разных нагрузках.
